# Explain — Deep dive into how code works

Explain the specified code, file, or system in detail:

1. **Read** the relevant code thoroughly
2. **Trace** the execution flow from entry point
3. **Identify** key patterns, data structures, and algorithms used
4. **Explain** in plain language:
   - What does this code do? (high level)
   - How does it work? (step by step)
   - Why was it likely written this way? (design decisions)
   - What are the key dependencies?
   - What are potential gotchas or edge cases?

Keep explanations clear and accessible. Use analogies where helpful. If the code is complex, build understanding progressively — start simple, add layers.
