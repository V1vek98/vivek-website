# Plan — Create an implementation plan

Create a detailed implementation plan for the task described:

1. **Understand** — Research the codebase to understand relevant code
2. **Explore options** — Consider 2-3 approaches, starting with the simplest
3. **Choose** — Pick the best approach and explain why
4. **Break down** — Split into concrete, ordered steps
5. **Save** — Write the plan to `docs/plans/[feature-name].md`

## Plan format:
```markdown
# Plan: [Feature Name]

## Goal
[One sentence describing what we're building]

## Approach
[Which approach we're taking and why]

## Steps
1. [ ] Step 1 — description
2. [ ] Step 2 — description
...

## Files to modify
- `path/to/file` — what changes

## Files to create
- `path/to/new/file` — what it does

## Testing strategy
- How we'll verify the implementation works

## Risks
- Potential issues and mitigations
```

Do NOT start implementing. Only plan.
